NTFS

Utilizado principalmente en sistemas Windows.

Permite el cifrado de archivos y control de acceso basado en permisos.

Admite tamaños de archivo y partición muy grandes.

Proporciona recuperación ante errores y metadatos avanzados.

FAT32

Compatible con varios sistemas operativos, incluidos Windows, macOS y Linux.

Es adecuado para dispositivos USB, tarjetas de memoria y otros medios extraíbles.

No admite archivos individuales mayores a 4 GB.

No ofrece características de seguridad avanzadas ni journaling.

exFAT

Desarrollado por Microsoft como una versión mejorada de FAT32.

Compatible con Windows, macOS y algunos sistemas Linux con controladores adicionales.

Ideal para dispositivos de almacenamiento externo, como unidades flash y discos duros externos.

Permite archivos de tamaño extremadamente grande y no tiene restricciones de 4 GB.

HFS+

Utilizado en sistemas macOS anteriores a macOS 10.13.

Admite metadatos extendidos, journaling y encriptación de archivos.

Proporciona soporte para nombres de archivo unicode y tamaños de archivo y partición grandes.

No es compatible de forma nativa con otros sistemas operativos.

APFS

Introducido en macOS 10.13 y posteriormente utilizado como el sistema de archivos predeterminado.

Diseñado específicamente para dispositivos de almacenamiento SSD y otros dispositivos de Apple.

Ofrece características avanzadas como journaling, instantáneas, encriptación y duplicación de archivos.

Proporciona un mejor rendimiento y una mayor eficiencia en comparación con HFS+.

EXT4

Utilizado en muchos sistemas operativos basados en Linux.

Admite tamaños de archivo y partición muy grandes.

Proporciona journaling para una recuperación rápida ante errores y confiabilidad del sistema.

Es compatible con atributos extendidos y encriptación a nivel de archivo.
